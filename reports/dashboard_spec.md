# Спецификация dashboard

Dashboard состоит из двух страниц: управленческий обзор и аналитический обзор. Все показатели строятся на витринах DuckDB из `data/marts/`.

## Страница 1. Управленческий обзор

Цель страницы — быстро показать состояние продуктовой воронки, выручку, потери на ключевых шагах и зоны для управленческого внимания.

### KPI cards

- Конверсия просмотра в покупку.
- Конверсия корзины в покупку.
- Повторная покупка за 30 дней.
- GMV.
- Доля брошенных корзин.

### Funnel chart

График показывает общий путь пользователя:

```text
Просмотр -> Корзина -> Покупка
```

Для каждого шага нужны:

- количество событий;
- доля перехода к следующему шагу;
- итоговая конверсия просмотра в покупку.

### Breakdown by category и brand

Блок сравнивает категории и бренды по:

- просмотрам;
- добавлениям в корзину;
- покупкам;
- выручке;
- конверсии просмотра в покупку.

Рекомендуемый формат: таблица с сортировкой и горизонтальная bar chart для top-15 категорий или брендов по трафику.

### Alert block: сегменты с самым высоким drop-off

Блок должен подсвечивать категории и бренды, где есть заметный трафик, но слабая конверсия в покупку.

Логика:

- взять категории и бренды с достаточным количеством просмотров;
- рассчитать конверсию просмотра в покупку;
- отсортировать по худшей конверсии;
- показать потенциальные причины для проверки: цена, карточка товара, наличие товара, качество трафика, ассортимент.

## Страница 2. Аналитический обзор

Цель страницы — дать продуктовой и аналитической команде детали для диагностики удержания, глубины сессии и приоритизации сессий по вероятности покупки.

### Cohort heatmap

Heatmap строится по `mart_retention`:

- строки: месяц первой покупки;
- столбцы: номер месяца после первой покупки;
- значение: retention rate.

Подписи:

- заголовок: `Когортное удержание по месяцу первой покупки`;
- ось X: `Месяцев после первой покупки`;
- ось Y: `Когорта`.

### Retention curve

Линейный график показывает средний retention по номеру месяца после первой покупки.

Подписи:

- заголовок: `Кривая удержания покупателей`;
- ось X: `Месяцев после первой покупки`;
- ось Y: `Retention rate`.

### Scatter session depth vs conversion

Scatter plot показывает связь глубины сессии и конверсии:

- X: количество событий в сессии;
- Y: вероятность покупки или фактическая доля сессий с покупкой;
- размер или цвет: количество сессий в группе.

Группировка может быть по бинам `events_cnt`.

### Table: top categories by traffic with worst conversion

Таблица показывает категории с высоким трафиком и слабой конверсией.

Поля:

- категория;
- просмотры;
- корзины;
- покупки;
- конверсия просмотра в покупку;
- выручка.

### Decile chart по модели вероятности покупки

График показывает долю покупок по децилям скоринга модели.

Ожидаемая интерпретация:

- верхний дециль должен иметь более высокую долю покупок, чем среднее по выборке;
- показатель используется для приоритизации аналитических гипотез;
- это scenario analysis, а не доказательство причинного эффекта.
